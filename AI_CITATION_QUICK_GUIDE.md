# AI 引用来源快速指南

## 🚫 禁止事项

### 1. 不要编造虚假来源
```
❌ 错误：提供不存在的网站
❌ 错误：编造虚假的文章标题
❌ 错误：创造虚假的引用内容
```

### 2. 不要在表格中使用引用标记
```markdown
❌ 错误示例：
| 地区 | 数据 |
|------|------|
| 山东 | 552.9 [1] |  ← 会破坏表格渲染

✅ 正确示例：
根据最新报告 [1]，各地区数据如下：

| 地区 | 数据 |
|------|------|
| 山东 | 552.9 |
| 重庆 | 426.8 |
```

### 3. 不要使用不完整的 URL
```
❌ 错误：www.example.com
❌ 错误：example.com/page
✅ 正确：https://www.example.com/page
```

## ✅ 推荐做法

### 1. 有真实来源时
```
草莓的最佳生长温度是 15-25°C [1]。

<SOURCES>
[{
  "number": "1",
  "title": "草莓栽培技术 - 农业部",
  "url": "https://www.moa.gov.cn/...",
  "description": "官方栽培指南",
  "quote": "最适温度为15-25°C"
}]
</SOURCES>
```

### 2. 无确定来源时
```
草莓的最佳生长温度是 15-25°C。
这是基于农业知识库的一般性建议。
```

### 3. 表格数据引用
```markdown
以下是各地区数据 [1]：

| 地区 | 指数 |
|------|------|
| 山东 | 552.9 |
| 重庆 | 426.8 |

注：数据来源于农业统计年鉴 [1]。
```

## 📋 引用格式检查清单

使用引用前，请确认：
- [ ] URL 是完整的（包含 https://）
- [ ] 网站真实存在且可访问
- [ ] 内容描述准确
- [ ] 引用文字真实存在
- [ ] 没有在表格内使用 [n]
- [ ] 引用编号连续且正确

## 🎯 核心原则

1. **真实性第一**：宁可不提供来源，也不编造
2. **格式正确**：避免破坏 Markdown 渲染
3. **诚实透明**：不确定时明确说明
4. **用户体验**：确保信息清晰易读

## 💡 常见场景

### 场景 1：提供统计数据
```markdown
✅ 正确：
2023年草莓产量达到300万吨 [1]。

❌ 错误（表格内）：
| 年份 | 产量 |
|------|------|
| 2023 | 300万吨 [1] |  ← 不要这样做
```

### 场景 2：引用多个来源
```markdown
✅ 正确：
草莓种植面积逐年增加 [1][2]，预计未来还会继续扩大 [3]。

<SOURCES>
[
  {"number": "1", "title": "...", "url": "https://..."},
  {"number": "2", "title": "...", "url": "https://..."},
  {"number": "3", "title": "...", "url": "https://..."}
]
</SOURCES>
```

### 场景 3：技术文档引用
```markdown
✅ 正确：
根据官方文档 [1]，配置方法如下：

\`\`\`json
{
  "setting": "value"
}
\`\`\`

注：详细说明请参考官方文档 [1]。
```

## 🔍 自检问题

在提供引用前，问自己：
1. 这个网站真的存在吗？
2. 我能确认这个 URL 可以访问吗？
3. 引用的内容真的在这个页面上吗？
4. 我是否在表格内使用了引用标记？
5. 如果不确定，我是否应该不提供来源？

## 📞 需要帮助？

如果遇到以下情况：
- 不确定来源是否真实
- 无法找到合适的引用
- 表格渲染出现问题

**解决方案**：
- 不使用 `<SOURCES>` 标签
- 明确说明这是基于知识库的回答
- 建议用户查阅权威资料

---

**记住**：质量胜于数量，真实胜于虚假！
